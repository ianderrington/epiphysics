# Phase 3: Design & Architecture

## Overview

Define system architecture, security, compliance, and **formal domain structure**.

## Key Activities

### 1. Define Architectural Domains (REQUIRED)

**Before designing features, define formal domains.**

Domains represent architectural subsystems and must be documented in:
- `supernal.yaml` (configuration)
- `docs/architecture/domains/README.md` (documentation)
- ADR (architecture decision record)

**See**: [SOP-T.09: Domain Management](../tools/SOP-T.09-domain-management.md)

### 2. Compliance Requirements

Document compliance needs and regulatory requirements.

**See**: [SOP-3.03: Compliance Requirements](SOP-3.03-compliance-requirements.md)

### 3. Security Analysis

Perform security threat modeling and risk assessment.

**See**: [SOP-3.04: Security Analysis](SOP-3.04-security-analysis.md)

## Outputs

- [ ] Architectural domains defined in `supernal.yaml`
- [ ] Domain documentation created
- [ ] Compliance requirements documented
- [ ] Security analysis complete
- [ ] ADRs created for major decisions

## Next Phase

**Phase 4: Planning** - Break down features into tasks (assigned to domains)

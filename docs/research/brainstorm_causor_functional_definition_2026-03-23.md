## Prompt captured (plain)

Need clearer language in Part 1.5:

- Causor should read plainly as **"that which causes"** (functional definition).
- Causors should be described as functional operators that produce state change.
- Language should avoid ambiguity and make intuitive sense immediately.

Also requested:

- better definition framing of **causal density**,
- explicit attention to labeling/separability (what distinguishes X from non-X),
- recognition that better labeling may improve prediction,
- consider possible dualism between causal nature and naming/labeling layer.

## Candidate wording direction for Part 1.5

### Functional definition candidate

> A **causor** is the minimal functional unit that can produce a state change in a system — i.e., *that which causes*.

### Operator framing candidate

> Causors can be treated as causal operators over state: each causor specifies a transformation tendency from one local state configuration to another.

### Plain-language bridge sentence

> If a component cannot, even in principle, make a difference to what happens next, it is not functioning as a causor in this model.

## Causal density refinement directions

1. Distinguish **structural causal density** (potential causal pathways) from **active causal density** (pathways currently firing).
2. Separate **internal density** from **boundary-crossing density**.
3. Make auto-causal density explicitly loop-level and emergent (already partly in 1.5; tighten wording).

## Labeling + separability hypothesis (to research)

Hypothesis:

> Better labels (cleaner boundaries between entity and non-entity, and between operator-types) improve predictive performance because they reduce representational ambiguity in causal attribution.

Potential formal handle:

- boundary quality as an information criterion,
- compare prediction error under coarse vs refined entity/causor labels,
- connect to causal emergence / effective information style metrics.

## Open questions to answer

1. Is “causor” best retained, or should subtitle include “causal operator” for clarity?
2. How to define causal density so it is measurable across physical/institutional/cognitive domains?
3. What is the minimal boundary/separability condition for an entity in Epimechanics?
4. How does label quality quantitatively affect prediction quality?
